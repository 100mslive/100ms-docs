## Introduction

Templates determine behaviour of roles in a particular room.  They give you flexibility to determine behaviour of different roles on different devices. 

The template you set while creating room is available to in the callback of `init` API while joining the room. You can determine person's behaviour in the room based on parameters set while creating the template. 

## Components of a template

A template contains the roles that the template should have. Roles contains properties of a role which any joining peer can take.

- `name`: `<template name>`
- `customer`: `<your customer id>`.
- `roles` : array of roles in the template.

Components of a role:

- `name` : name of the role
- `publishParams` : publish parameters
    - `allowed` :array containing - audio/video/screen showing which to allow and which to not.
    - `audio` : Audio parameters - bitrate & codec.
    - `video` : video parameters - bitrate, codec, framerate, width & height.
    - `screen` : screen parameters - bitrate, codec, framerate, width & height.

<Image width={1726} height={564} alt="Create Template" src="/docs/v2/template-1.png" />


- `SubscribeParams`
    - `subscribeToRoles` : array of roles to subscribe to.
    - `maxSubsBitRate` : max subscription bitrate

## Getting templates on dashboard

To get template from dashboard, visit - [https://dashboard.100ms.live/templates](https://dashboard.100ms.live/templates)

From dashboard, you can add, modify, delete roles of a template. You can use the template from dashboard for creating rooms in Create Room API section.
