## Overview

To allow users to join a 100ms video conferencing session inside your app, you first need to create a `room`.

## Create Room On Dashboard

You can create room on [100ms Dashboard](https://dashboard.100ms.live/create-rooms). While creating a room you can specify it's name, roles or enable recording.

<Image width={2168} height={1212} alt="Create a room" src="/docs/v2/create-a-room.png" />

You can add or modify roles on [100ms Dashboard](https://dashboard.100ms.live/roles).

## Create Room using API

You can use following API to create room.

### Headers

| media type                     | type     | response type                |
| ------------------------------ | -------- | ---------------------------- |
| Content-type <br/> (REQUIRED)  | `string` | application/json             |
| Authorization <br/> (REQUIRED) | `string` | Bearer `<Server Side Token>` |

### Body Paramters

| params                          | type     | description                                                                                                                                                                                                                                                                                                          |
| ------------------------------- | -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| name <br/> (REQUIRED)           | `string` | <br/> A unique identifier you can assign to 100ms rooms. This is `case-insensitive`. Accepted characters are `a-z, A-Z, 0-9, and . - : _` If not provided, this is generated automatically. <br/><br/> Please note that with `createRoom` is called with the same room name, then the same room ID is returned <br/> |
| description <br/> (OPTIONAL)    | `string` | A string to describe your room's usage - eg. "9PM English Class Batch 2"                                                                                                                                                                                                                                             |
| template <br/> (OPTIONAL)       | `string` | You can get template name from room details section on dashboard                                                                                                                                                                                                                                                     |
| recording_info <br/> (OPTIONAL) | `object` | Object of type `recording_info`. This object contains information for enabling recording/setting storage location for recordings etc.                                                                                                                                                                                |
| region <br/> (OPTIONAL)         | `string` | Region in which you want to create room - `in` - India region, `us` - US region or `auto` which stands for automatic region selection which happens also in case this parameter is not provided |

### recording_info

| Name        | Type      | Description                                                                                                                                                                                     | Required |
| ----------- | --------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------- |
| enabled     | `boolean` | Enable recording. Disabled by default                                                                                                                                                           | no       |
| upload_info | `object`  | object of type `upload_info`. This object contains information on recordings storage location. If you want to store recording with 100ms, and not use your own s3, don't add this to the object | no       |

### upload_info

| Name        | Type     | Description                                                                                                                                                                                      | Required |
| :---------- | :------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :------- |
| type        | `string` | Upload Destination type. Currently, only `s3` is supported                                                                                                                                       | Yes      |
| location    | `string` | Name of the AWS s3 bucket in which you want to store all recordings                                                                                                                              | Yes      |
| prefix      | `string` | Upload prefix path                                                                                                                                                                               | No       |
| options     | `object` | Options to be used for uploading i.e. region, ACLs etc                                                                                                                                           | No       |
| credentials | `object` | Object of type `Credentials`. This is used to share AWS credentials to access the s3 bucket specified. Leave this blank if you are using an `IAM role` to provide 100ms access to your s3 bucket | No       |

### credentials

| Name   | Type     | Description                                                                  | Required |
| :----- | :------- | :--------------------------------------------------------------------------- | :------- |
| key    | `string` | Access Key for the AWS account hosting the s3 bucket for recordings' storage | Yes      |
| secret | `string` | Secret for the AWS account hosting the s3 bucket for recordings' storage     | Yes      |
