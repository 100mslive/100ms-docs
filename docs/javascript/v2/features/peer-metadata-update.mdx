Want to add features like raiseHand, emoji reactions etc. or want to change the name of peer.

## Changing Peer Metadata

These are the options which need to be passed to change peer metadata
1. **peerID**: *HMSPeerID*. The local peer id whose metadata is to be changed.
2. **metadata**: *string*. A stringified object containing the peer metadata.


```js
try {
    const peerID = hmsStore.getState(selectLocalPeerID);
    const metadata = JSON.stringify({isHandRaised: true});
    hmsActions.changeMetadata(peerID, metadata);
} catch(error) {
    // Invalid Peer ID or not connected to room etc.
    console.error(error);
}
```

## Changing Name of local peer.

These are the options which need to be passed to change peer's name.
1. **peerID**: *HMSPeerID*. The local peer id whose name is to be changed.
2. **name**: *string*. New name of the local peer.


```js
try {
    const peerID = hmsStore.getState(selectLocalPeerID);
    const name = "Michael Scott";
    hmsActions.changeName(peerID, name);
} catch(error) {
    // Invalid Peer ID or not connected to room etc.
    console.error(error);
}
```

## Peer Metadata update Notification

Once the local peer changes the metadata, the remote peers will receive a notification
with type METADATA_UPDATED and [HMSPeer](../../../api-reference/javascript/v2/interfaces/HMSPeer)
object as notification data.


```js
hmsNotifications.onNotification((notification) => {
    if (!notification) {
        return;
    }

    switch (notification.type) {
        // ...Other notification type cases
        case 'METADATA_UPDATED':
            // Redirect or Show toast to user
            toast(notification.data.metadata);
            break;
    }
});
```
