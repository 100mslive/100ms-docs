Track Events can be used to understand the participants activity during the session, It would get you the reference points
in time ex. when user have unmuted, shared screen, stopped video. Every event has a `mute` property under `track` to determine mute status at the time of event.
There are three events currently being published -
#### track.add.success
Event is published when track is added, it is represents start of track lifecycle.
#### track.update.success
Event is published mainly when user has unmuted/muted.
#### track.remove.success
Event is published when the track has ended, has `stopped_at` property as exact time of stopping (should be very close to event timestamp).

<GetRequest title="https://api.100ms.live/v2/analytics/events" />
<Request id="req-0">

```bash
curl --location --request GET 'https://api.100ms.live/v2/analytics/events' \
--header 'Authorization: Bearer <management_token>'
        ```

</Request>

## Allowed Filters
To be specified as query parameters

| Name       | Type     | Description                                                                                                                                   | Required |
| ---------- | -------- |-----------------------------------------------------------------------------------------------------------------------------------------------| -------- |
| type       | `string` | Type of event, can be specified multiple times (valid values `track.add.success`, `track.update.success`, `track.remove.success`)             | yes      |
| room_id    | `string` | Filter events for a room (mandatory)                                                                                                          | yes      |
| session_id | `string` | Filter events for a room's session                                                                                                            | no       |
| peer_id    | `string` | Filter peer's events by `peer_id`                                                                                                             | no       |
| user_id    | `string` | Filter peer's events by `user_id`                                                                                                             | no       |
| limit      | `int`    | Limits number of events in result (Default:`10`, Max: `100`)                                                                                  | no       |
| start      | `string` | Specify `next` in previous response as `start` to get next set of result                                                                      | no       |

### Track Event Object

| Attribute  | Description                                                                                 |
| ---------- | ------------------------------------------------------------------------------------------- |
| event      | Event name (values: `track.add.success`, `track.update.success`, `track.remove.success`)    |
| timestamp  | Event timestamp                                                                             |
| id         | Event id                                                                                    |
| room_id    | Event room id                                                                               |
| session_id | Event session id                                                                            |
| room_name  | Event room name                                                                             |
| peer       | Peer details as `id` (peer id), `user_id` (customer specified), `name`, `joined_at`, `role` |
| track      | Track object (details below)                                                                |

### Track Object

| Attribute  | Description                                                |
| ---------- | ---------------------------------------------------------- |
| id         | Track ID unique for a peer                                 |
| stream_id  | Reference to internal stream object                        |
| type       | Type of track (values: `audio`, `video`)                   |
| source     | Source of input (values: `regular`, `screen`)              |
| mute       | Boolean value specifying if track is muted                 |
| started_at | Track start time                                           |
| stopped_at | Track stop time (only specified in `track.remove.success`) |
